# NTS-1 野外配信向けステレオコンプレッサ

[English version / 英語版はこちら](README.md)

DJI Mic MiniやプリアンプをつけたステレオECMなどを、より高音質にライブ配信するための
KORG NTS-1(初代)用MODプラグイン(MODFXモジュール)のセットです。

## モジュール一覧

### stcomp — ステレオコンプリミッター

野外配信に最適なステレオコンプリミッターです。DJI Mic MiniとAMS-22を組み合わせることで、
野外で高音質な配信が可能です。

**操作**

- **TIME (A):** コンプレッション度。時計回りにスレッショルド -6 dBFS から -36 dBFS。
- **DEPTH (B):** コンプレッサー後段のメークアップゲイン。0 から +40 dB。

**固定値**

- Ratio: 固定 4:1
- Attack: 固定 5 ms
- Release: 固定 150 ms
- ステレオ検出: `max(abs(L), abs(R))`。左右チャンネルへ同じゲインを適用
- 出力リミッター: ルックアヘッド無し、-1 dBFS シーリング、リリース 50 ms

### agccomp — オートゲイン + コンプ/リミッター

入力レベルを目標ラウドネス **約 -14 LUFS** へ自動追従させてから、コンプレッサーと
リミッターを通す、手放し運用向けのバリエーションです。`stcomp` のメークアップゲインは
廃止し、代わりにAGC段が最適なレベルへ調整します。

信号連鎖: **AGC → コンプレッサー → リミッター**

**操作**

- **TIME (A):** AGCの反応時間。時計回りに 1 秒から 30 秒 (対数)。
- **DEPTH (B):** 未使用。

**固定値**

- AGC目標: 約 -14 LUFS (K特性を省略したBS.1770の近似)
- AGCゲイン範囲: -24 dB から +40 dB。-45 dBFS RMS 未満では凍結 (無音ゲート)
- AGCラウドネス測定窓: 400 ms (モーメンタリ相当)
- コンプレッサー: スレッショルド -18 dBFS 固定、レシオ 4:1、アタック 5 ms、リリース 150 ms
- 出力リミッター: ルックアヘッド無し、-1 dBFS シーリング、リリース 50 ms
- ステレオ検出: `max(abs(L), abs(R))`。左右チャンネルへ同じゲインを適用

### fieldcomp — LEVEL + オートマークアップ コンプ/リミッター

DJI Mic Miniのような低レベル入力向けの構成です。手動の **LEVEL** つまみで入力を
大まかに持ち上げ、コンプレッサーの **オートマークアップゲイン** が出力を約
**-14 LUFS** へ精密に合わせます。合計で最大 +40 dB (LEVEL +20 dB + マークアップ
+20 dB) の増幅が可能です。`agccomp` と違い、LEVEL つまみでコンプレッサーの
効き具合を能動的に決められます。

信号連鎖: **LEVEL → コンプレッサー → オートマークアップ → リミッター**

**操作**

- **TIME (A):** オートマークアップの反応時間。時計回りに 1 秒から 10 秒 (対数)。
- **LEVEL (B):** 手動の入力ゲイン。0 dB から +20 dB。

**固定値**

- オートマークアップ目標: 約 -14 LUFS、ゲイン範囲 0 から +20 dB (減衰はしない)
- オートマークアップはコンプ後を測定。-45 dBFS RMS 未満では凍結 (無音ゲート)
- コンプレッサー: スレッショルド -18 dBFS 固定、レシオ 4:1、アタック 5 ms、リリース 150 ms
- 出力リミッター: ルックアヘッド無し、-1 dBFS シーリング、リリース 50 ms
- ステレオ検出: `max(abs(L), abs(R))`。左右チャンネルへ同じゲインを適用

## ビルド

ビルドには Docker, git, make などが必要です。`logue-sdk` とそのDockerイメージは
初回ビルド時に自動的に取得・ビルドされます。

```sh
# 全MODFXモジュールを dist/ へビルド
$ make

# 特定のモジュールのみビルド
$ make stcomp

# ビルド成果物を削除 (logue-sdk は残す)
$ make clean

# logue-sdk も含めて完全に削除
$ make distclean
```

ビルドされた `*.ntkdigunit` は `dist/` に出力されます。

## MODFXの追加方法

1. モジュール名と同名のディレクトリ (例: `mymodfx/`) を作成します。
2. その中に `mymodfx.cpp` / `project.mk` / `manifest.json` を配置します。
3. `Makefile` の `MODFX_MODULES` にモジュール名を追記します。
